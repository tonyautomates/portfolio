AI automation

AI Operations Assistant

A workflow assistant designed to automate repetitive business tasks like inbox triage, quote drafting, lead sorting, status updates, and internal follow-ups without creating messy manual work.

Automation AI Workflows Operations

The main goal is to build a system that reduces small operational delays across a business. Instead of using AI only for chat, this project focuses on real workflow execution: reading incoming requests, classifying urgency, generating a draft response, logging the task, and pushing the right next step to a dashboard or team channel.

I want the assistant to support structured decision-making rather than random automation. That means using clear trigger conditions, approval checkpoints, fallback logic, and a record of what the system did so the operator can trust it and improve it over time.

Core functions

  • Classify inbound messages by topic, urgency, and required action.
  • Draft replies, meeting summaries, and short client updates.
  • Move tasks into a simple workflow with human approval where needed.
  • Generate a daily operations report with bottlenecks and next actions.

Why it matters

  • Reduces time lost to repeated admin work and context switching.
  • Keeps business communication more consistent and faster to respond.
  • Creates a repeatable process that can scale with more clients or projects.
  • Builds practical experience in AI systems that support real operations.

Planned build scope

The first version would connect a form, email inbox, or message feed into a lightweight processing pipeline. From there the system would tag the request, create a draft output, store the result, and wait for operator approval before sending anything important. Later versions could include customer memory, CRM syncing, and better reporting on time saved per week.

AI anti-drone systems

Drone Threat Detection Platform

A concept platform for detecting, classifying, and tracking suspicious drone activity by combining sensor feeds, visual analysis, alert logic, and operator review into one clearer control layer.

Computer Vision Defense Tech Sensors

This project is focused on the software side of modern anti-drone systems: how to interpret incoming signals quickly, reduce false alarms, and give the operator a better picture of what is happening. The system is not just about seeing a drone, but about deciding whether it is normal, risky, or an active threat that needs escalation.

The platform would fuse camera input, radar-style detection data, acoustic cues, and rule-based alert thresholds into a single interface. AI would be used to prioritize likely threats, highlight unusual movement patterns, and support faster human decisions instead of flooding the operator with noise.

System focus

  • Multi-sensor event fusion for more reliable detection than a single source.
  • Visual classification of drone type, flight behavior, and probable intent.
  • Threat scoring based on route, altitude, speed, and protected zone proximity.
  • Operator dashboards that show what matters first instead of raw data overload.

Engineering challenges

  • Reducing false positives from birds, weather, noise, or irrelevant motion.
  • Keeping latency low enough for real-time response decisions.
  • Making the model explain its alerts in a useful way for human operators.
  • Designing a system that stays useful even when one sensor feed is weak.

What I want to explore

I want to understand how practical defense-oriented AI systems are designed from a systems thinking perspective, especially how detection, classification, alerting, and decision support work together. Even at concept stage, this project helps me think more seriously about reliability, safety, interfaces, and how software can support complex technical environments.

3D printing

Functional 3D Printing Lab

A practical project focused on designing and printing useful parts, test fixtures, and small custom components that solve real problems instead of producing decorative prints only.

CAD Prototyping Fabrication

The idea behind this project is to treat 3D printing as an engineering tool. Instead of printing random objects, I want to build a process for identifying a physical problem, sketching a solution, designing a part, testing fit and strength, and refining the print until it becomes genuinely useful.

The project would include simple jigs, cable guides, machine brackets, sensor mounts, storage inserts, and other functional pieces that support workshop, automation, or electronics work. Over time it could become a small library of repeatable designs with print notes, failure lessons, and material choices.

Build direction

  • Design parts for actual use cases instead of one-off visual experiments.
  • Track tolerances, material strength, and print orientation decisions.
  • Create iteration notes so each version improves based on evidence.
  • Combine design thinking with hands-on testing and workshop practicality.

Outputs I want

  • A portfolio of functional prints with clear before-and-after problem solving.
  • Better understanding of how CAD decisions affect real physical performance.
  • Reusable parts for electronics, automation, and general workspace organisation.
  • A more disciplined product mindset around prototyping and improvement.

Longer-term value

This project helps connect digital design with physical execution. It supports problem solving, fabrication skills, and product thinking at the same time. It also fits naturally with future AI and automation work, because many technical builds eventually need custom mounts, cases, adapters, or testing tools that are easiest to prototype with 3D printing.
